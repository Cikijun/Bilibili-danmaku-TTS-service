弹幕TTS服务使用说明
项目简介
弹幕TTS服务是一款面向B站直播间的智能语音播报工具，支持欢迎观众、弹幕播报、点赞/礼物/舰长/醒目留言感谢等场景的语音播报，同时可选配AI互动功能，无需修改代码，仅通过配置文件即可完成全功能自定义。

声明：项目代码公开，但本人注册的开放平台密钥不予公开。

快速上手
1. 配置文件准备
工具核心配置文件为config.ini，编码格式为UTF-8，首次启动需要输入身份码，可在直播姬的幻星互动里获取，输入身份码后按回车，会自动生成config.ini配置文件，保存在程序同目录下。同时会创建bili_live_tts.log日志文件，程序不会主动删除日志文件，需手动删除。

2. 核心配置（必填）
打开config.ini，找到[CORE_CONFIG]模块，填写从B站互动平台获取的auth_code（身份码）：
[CORE_CONFIG]
auth_code = 你的B站互动平台身份码（如F4DYNSUPZGN40）
tts_rate = +0%
tts_volume = +0%
tts_voice = zh-CN-XiaoyiNeural

3. 功能开关（可选）
根据需要开启/关闭对应功能（0=关闭，1=打开），默认全部开启：
[FUNCTION_SWITCH]
welcome_enable = 1
dm_enable = 1
like_broadcast_enable = 1
gift_thank_enable = 1
guard_enable = 1
super_chat_enable = 1

4. 自定义播报文案（可选）
可修改各场景的播报文案，保留占位符（如name、msg）即可自动替换动态信息：
[CUSTOM_MSG]
welcome_msg = 欢迎name进入直播间！
dm_msg = name说：msg
like_thank_msg = 感谢name点了count个赞！
gift_thank_msg = 感谢name赠送gift_num个gift_name！
guard_msg = 感谢name开通了guard_month个月的guard_name！
super_chat_msg = 感谢name发送的SuperChat！：message

5. AI互动配置（可选）
如需启用AI互动功能，需填写AI平台相关信息，默认关闭（ai_enable=0）：
[AI_CONFIG]
ai_enable = 1  # 开启AI功能请改为1
ai_api_base = 你的AI平台接口地址
ai_api_key = 你的AI平台接口密钥
ai_model = 你要使用的AI模型名称
ai_temperature = 0.7
ai_system_prompt = 自定义AI人设/互动规则

配置项详细说明（含功能与用法）
 [CORE_CONFIG] 核心配置（控制工具基础运行与语音输出）
1. 配置项：auth_code
   功能：工具接入B站直播间的身份验证凭证，无此参数工具无法关联直播间
   用法：必须从B站互动平台申请获取，复制完整身份码填入，不可为空
   必填：是
   默认值：无
   示例/备注：F4DYNSUPZGN40（示例值，实际需替换为个人身份码）

2. 配置项：tts_rate
   功能：调整TTS语音的播放速度，满足不同播报节奏需求
   用法：需带+/-符号，0%需严格写为+0%；数值为正加快语速，为负减慢语速
   必填：否
   默认值：+0%
   示例/备注：+10%（语速加快10%，适合快速播报弹幕）、-5%（语速减慢5%，适合清晰感谢礼物）

3. 配置项：tts_volume
   功能：调整TTS语音的输出音量，避免音量过高或过低影响直播体验
   用法：需带+/-符号，0%需严格写为+0%；数值为正提高音量，为负降低音量，建议调整范围-50%~+50%
   必填：否
   默认值：+0%
   示例/备注：+20%（音量提高20%，适合嘈杂环境）、-10%（音量降低10%，避免音量过大刺耳）

4. 配置项：tts_voice
   功能：选择TTS语音的音色，适配不同直播风格
   用法：需填写Edge TTS官方公布的有效音色名称，名称错误会导致语音播报失效
   必填：否
   默认值：zh-CN-XiaoyiNeural（元气少女音色，适配多数娱乐直播）

[FUNCTION_SWITCH] 功能开关（控制各场景播报功能的启用状态）
1. 配置项：welcome_enable
   功能：控制"新用户进入直播间"时的语音欢迎功能
   用法：填写0关闭该功能（新用户进入无语音提示），填写1打开该功能（新用户进入自动播报欢迎语）
   取值：0=关闭，1=打开
   默认值：1（默认开启欢迎功能，提升用户归属感）

2. 配置项：dm_enable
   功能：控制"观众发送弹幕"时的语音播报功能
   用法：填写0关闭该功能（弹幕仅文字显示，无语音播报），填写1打开该功能（弹幕发送后自动语音朗读）
   取值：0=关闭，1=打开
   默认值：1（默认开启弹幕播报，增强直播互动感）

3. 配置项：like_broadcast_enable
   功能：控制"观众点赞"时的语音感谢功能
   用法：填写0关闭该功能（观众点赞无语音反馈），填写1打开该功能（累计点赞达到一定数量后自动语音感谢）
   取值：0=关闭，1=打开
   默认值：1（默认开启点赞播报，鼓励观众互动）

4. 配置项：gift_thank_enable
   功能：控制"观众赠送礼物"时的语音感谢功能
   用法：填写0关闭该功能（观众送礼无语音感谢），填写1打开该功能（观众送礼后自动语音播报感谢语）
   取值：0=关闭，1=打开
   默认值：1（默认开启礼物感谢，提升观众送礼积极性）

5. 配置项：guard_enable
   功能：控制"观众开通舰长/守护"时的语音感谢功能
   用法：填写0关闭该功能（开通舰长/守护无语音反馈），填写1打开该功能（开通后自动语音播报专属感谢语）
   取值：0=关闭，1=打开
   默认值：1（默认开启舰长/守护感谢，重视高价值用户）

6. 配置项：super_chat_enable
   功能：控制"观众发送醒目留言（SuperChat）"时的语音播报功能
   用法：填写0关闭该功能（醒目留言仅文字显示），填写1打开该功能（醒目留言发送后自动语音朗读内容）
   取值：0=关闭，1=打开
   默认值：1（默认开启醒目留言播报，突出高价值互动）

[CUSTOM_MSG] 自定义文案（个性化各场景的语音播报内容）
1. 配置项：welcome_msg
   功能：定义新用户进入直播间时的欢迎语内容
   用法：可修改文字内容，必须保留占位符"name"（工具会自动将其替换为进入直播间的用户昵称）
   占位符说明：name=用户昵称
   默认值：欢迎name进入直播间！
   示例/备注：自定义示例："欢迎可爱的name宝来到直播间呀～"（保留name占位符即可正常替换）

2. 配置项：dm_msg
   功能：定义弹幕播报时的语音文案格式
   用法：可修改文字内容，必须保留占位符"name"（用户昵称）和"msg"（弹幕内容），否则无法显示完整信息
   占位符说明：name=用户昵称、msg=弹幕内容
   默认值：name说：msg
   示例/备注：自定义示例："收到name的弹幕：msg～"（符合AI人设的温柔表述）

3. 配置项：like_thank_msg
   功能：定义感谢观众点赞时的语音文案
   用法：可修改文字内容，必须保留占位符"name"（用户昵称）和"count"（点赞数量），确保信息准确
   占位符说明：name=用户昵称、count=点赞数量
   默认值：感谢name点了count个赞！
   示例/备注：自定义示例："谢谢name宝的count个赞，太给力啦～"

4. 配置项：gift_thank_msg
   功能：定义感谢观众赠送礼物时的语音文案
   用法：可修改文字内容，必须保留占位符"name"（用户昵称）、"gift_num"（礼物个数）、"gift_name"（礼物名称）
   占位符说明：name=用户昵称、gift_num=礼物个数、gift_name=礼物名称
   默认值：感谢name赠送gift_num个gift_name！
   示例/备注：自定义示例："超级感谢name宝送的gift_num个gift_name，爱你哟～"

5. 配置项：guard_msg
   功能：定义感谢观众开通舰长/守护时的语音文案
   用法：可修改文字内容，必须保留占位符"name"（用户昵称）、"guard_month"（开通时长）、"guard_name"（舰长/守护类型）
   占位符说明：name=用户昵称、guard_month=开通时长（月）、guard_name=守护/舰长类型
   默认值：感谢name开通了guard_month个月的guard_name！
   示例/备注：自定义示例："恭喜name宝开通guard_month个月的guard_name，成为直播间的守护神～"

6. 配置项：super_chat_msg
   功能：定义感谢观众发送醒目留言时的语音文案
   用法：可修改文字内容，必须保留占位符"name"（用户昵称）和"message"（醒目留言内容）
   占位符说明：name=用户昵称、message=醒目留言内容
   默认值：感谢name发送的SuperChat！：message
   示例/备注：自定义示例："收到name宝的SuperChat：message，谢谢支持呀～"

[AI_CONFIG] AI配置（控制AI互动功能的运行参数，默认关闭）
1. 配置项：ai_enable
   功能：AI互动功能的总开关，控制是否启用AI助手与观众互动
   用法：填写0关闭AI功能（仅保留基础TTS播报），填写1打开AI功能（AI助手自动回复观众弹幕、参与互动）
   必填：否
   默认值：0
   取值：0=关闭，1=打开
   示例/备注：无需AI互动时保持0即可，需要智能互动时改为1，同时需配置下方AI相关参数

2. 配置项：ai_api_base
   功能：指定AI模型的接口地址，工具通过该地址连接AI平台
   用法：AI功能开启时（ai_enable=1）必填，需填写完整接口地址（含端口和路径），地址错误会导致AI无法连接
   必填（AI开启时）：是
   默认值：无
   示例/备注：如百度智能云千帆平台接口地址（https://aip.baidubce.com/rpc/2.0/ai_custom/v1/wenxinworkshop/chat/ernie-4.0）、CloseAI代理地址（https://api.openai.com/v1）

3. 配置项：ai_api_key
   功能：AI平台的接口调用密钥，用于验证调用权限
   用法：AI功能开启时（ai_enable=1）必填，需从对应AI平台控制台申请获取，密钥泄露会导致接口被滥用，需妥善保管
   必填（AI开启时）：是
   默认值：无
   示例/备注：各AI平台密钥格式不同，需按平台要求完整复制（如sk-xxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx）

4. 配置项：ai_model
   功能：指定要使用的AI模型，不同模型的互动效果和响应速度不同
   用法：AI功能开启时（ai_enable=1）必填，需填写AI平台支持的模型名称，名称需与接口地址匹配
   必填（AI开启时）：是
   默认值：无
   示例/备注：如ERNIE-4.0（百度智能云）、Qwen-7B-Chat（阿里通义千问）、glm-4（智谱AI）、gpt-3.5-turbo（OpenAI）

5. 配置项：ai_temperature
   功能：控制AI生成回复的随机性和创造性，调整AI互动风格
   用法：取值范围0~1，无需手动修改时保留默认值即可；需要AI回复更固定时调低数值，需要更灵活时调高数值
   必填：否
   默认值：0.7（平衡随机性和准确性，适合日常互动）
   示例/备注：0.1（回复高度固定，仅按规则回应）、0.9（回复更有创造性，偶尔会有俏皮表达）

6. 配置项：ai_system_prompt
   功能：定义AI助手的人设、互动风格和回复规则，让AI适配直播间氛围
   用法：可根据直播风格修改，需明确AI的身份、语气、互动边界（如"禁止回复敏感话题"），默认人设为二次元白丝猫娘
   必填：否
   默认值：你是一个直播间助手，需要帮助主播和观众互动，你喜欢二次元，是个白丝猫娘，说话的时候略微有些胆小，且不擅长一次性说很多话。
   示例/备注：自定义示例："你是一个可爱的二次元主播助手，说话温柔俏皮，只回复与直播相关的互动内容，不讨论敏感话题，回复不超过30字"

配置修改注意事项
1. 调整tts_rate和tts_volume时，必须保留+/-符号，0%需严格写为+0%（如写成0%或无符号会导致配置失效）；
2. 音色名称（tts_voice）需严格匹配Edge TTS官方命名，可通过Edge TTS官方文档查询有效音色，错误名称会导致TTS语音无法播放；
3. 自定义文案中的占位符（如name、count、gift_num、msg等）不可删除或修改，否则工具无法自动替换用户信息、礼物信息等动态内容；
4. 功能开关（所有enable结尾的配置项）仅支持0或1两个取值，填写其他数值（如2、on、off）会导致功能异常；
5. AI接口密钥（ai_api_key）属于敏感信息，请勿分享给他人或公开，避免接口被恶意调用产生额外费用；
6. 配置文件修改后需保存，工具会实时读取最新配置，无需重启工具即可生效。

常见问题
 Q1：修改配置后不生效？
A1：首先确认配置文件编码为UTF-8（若为其他编码可能导致工具无法读取），且修改后已保存；其次检查配置项格式（如tts_rate是否带符号、功能开关是否为0/1、占位符是否完整）；最后关闭工具重新启动（少数情况下需重启加载配置）。

Q2：TTS语音无声音？
A2：第一步检查系统音频输出是否正常（如音箱、耳机是否通电且音量开启）；第二步确认tts_volume设置合理（避免为-100%）；第三步核实tts_voice是否为Edge TTS官方有效音色名（可替换为默认音色测试）；第四步检查功能开关是否开启（如dm_enable是否为1）。

Q3：AI功能无法使用？
A3：首先确认ai_enable已设为1；其次检查ai_api_base、ai_api_key、ai_model是否填写正确（接口地址需带完整路径，密钥无多余空格，模型名称与平台匹配）；然后核实网络是否能正常访问AI平台接口（可通过浏览器访问接口地址测试连通性）；最后检查AI平台是否有足够的调用额度（额度不足会导致接口调用失败）。

Q4：弹幕播报重复或漏报？
A4：重复播报可能是功能开关被重复启用（如误开启多个同类工具），需关闭其他冲突工具；漏报可能是网络延迟或弹幕频率过高，可适当调整直播网络环境，或暂时关闭dm_enable功能减少负载。
